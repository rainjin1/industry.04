미션 알람 시스템
1. 프로젝트 개요
목표

소리만 울리는 단순한 알람이 아닌, 사용자가 완전히 잠에서 깰 수 있도록 행동을 요구하는 미션을 결합한 하드웨어 구현

주요 기능
OLED와 가변저항, 스위치를 사용한 시간 설정
delay의 한계를 벗어나기 위해 비동기 함수를 이용한 알람음 연주 + 비밀번호 입력 미션 동시 수행
제스처 센서를 사용한 최대 3회의 스누즈(Snooze) 기능
PSD 적외선 거리 센서를 이용한 입력 가능 범위 제한 (너무 가깝거나 멀 경우 입력 불가)
최종 소음 측정을 통한 알람 종료 및 날짜, 환경 정보 브리핑
2. 기술 스택
프로그램 언어
Python
하드웨어
PYC Basic (라즈베리파이 4, Soda OS 기반)
OLED(디스플레이), PiezoBuzzer(부저), PSD(적외선 거리 센서)
Gesture 센서, Sound 센서
LED 모듈(0~7번), 스위치(0, 1번)
Libraries
threading: 알람음, 제스처 인식, 비밀번호 입력을 병렬 처리하기 위함
pop: 하드웨어 추상화 계층(HAL) 라이브러리를 통한 센서 및 액추에이터 제어
random, time: 정답 생성, 비밀번호 점멸을 위한 랜덤 좌표 및 타이머 로직
개발 환경
JupyterLab: 인터렉티브 코드 테스트 및 실시간 센서 디버깅
3. 시스템 아키텍처
A. Setup Phase
스위치를 활용한 직관적인 시간 설정 (OLED display UI)
B. Wait Phase
운영 타이머 가동
C. Mission Phase (Core)

알람음 연주와 동시에 3가지 미션 수행

비밀번호 입력
PSD 센서로 적정 거리 유지
디스플레이에 점멸하는 4자리 비밀번호 입력
스누즈 (Snooze)
제스처 센서 (Left → Right) 인식 시 타이머 5분 연장 (최대 3회)
LED 가이드
스레드를 이용한 시각적 사용자 유도
D. Final Phase
소음 센서를 통한 "소리 질러 알람 끄기"
이후 날짜 및 환경 정보 브리핑
4. 핵심 기술 포인트
비동기 멀티태스킹 기반 사용자 경험(UX) 설계
알람음(PiezoBuzzer), 비밀번호 입력 감시(Switches, PSD), 제스처 인식(Gesture), 시각적 스누즈 가이드(LEDs)를 4개의 독립적인 스레드로 병렬 실행
스레드 간 충돌 방지를 위해 플래그 기반으로 스레드 생명주기 관리
각 워커 스레드 내 조건부 로직을 설계하여 불필요한 자원 사용을 줄이고, 메인 스레드와 동기화 진행
주피터랩 기반 실시간 센서 디버깅 및 수치 최적화
센서 원시 데이터를 실시간으로 모니터링
축적된 데이터를 기반으로 적절한 임계값 설정
평균값 기반 처리 적용 → 튀는 값을 그대로 반영하지 않도록 안전장치 구성
5. 트러블 슈팅
문제 1. CPU 점유율 증가 및 스레드 중복 실행

문제

여러 스레드가 동시에 실행되며 CPU 점유율이 과도하게 증가
동일 스레드가 중복 실행되는 문제 발생

해결

모든 스레드의 while 조건을 alarm_on으로 통일
루프 내부 첫 줄에 조건문을 추가하여 필요 없는 경우 즉시 continue 처리
스레드 실행 조건을 더 정교하게 설정하여 중복 실행 방지
문제 2. OLED UI 충돌

문제

PSD 거리 안내와 비밀번호 입력 UI가 동시에 렌더링되며 충돌 발생

해결

우선순위 기반 화면 갱신 로직 적용
다양한 센서 피드백이 동시에 출력될 때 발생하는 시각적 노이즈 제거
6. 프로젝트 회고

파이썬의 멀티스레딩 개념을 실제 하드웨어에 적용해보면서, 변수 제어와 스레딩 조건을 더 명확하게 설계해야 한다는 점을 느꼈습니다.

특히 소프트웨어에서는 어느 정도 동작하던 구조도, 하드웨어 환경에서는 센서와 출력 장치가 병렬로 움직이면서 병목 현상이 자주 발생했습니다.

이 과정에서 변수 상태 관리와 실행 타이밍을 더 정교하게 다루게 되었고, 시스템 전체 흐름을 고려하는 방식으로 접근하게 되었습니다.

또한 JupyterLab 환경을 활용하면서 하드웨어 디버깅 시간을 크게 줄일 수 있었고, 데이터 기반으로 값을 조정하는 경험을 할 수 있었습니다.

이번 프로젝트를 통해, 실제 환경에서는 시행착오가 곧 비용으로 이어질 수 있기 때문에 기획 단계에서 더 체계적으로 설계하는 것이 중요하다는 점을 느꼈습니다.
