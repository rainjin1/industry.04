미션 알람 시스템 (Smart Mission Alarm)
1. 프로젝트 개요
목표: 소리만 울리는 단순한 알람이 아닌, 사용자가 완전히 잠에서 깰 수 있도록 행동을 요구하는 미션을 결합한 하드웨어 구현.

주요 기능:

OLED/가변저항/스위치: 직관적인 시간 설정 UI 제공.

비동기 미션 수행: delay의 한계를 극복하기 위해 멀티스레딩을 적용, 알람음 연주와 비밀번호 입력을 동시에 진행.

제스처 스누즈: 제스처 센서를 활용한 최대 3회의 스누즈(Snooze) 기능.

입력 거리 제한: PSD 적외선 센서로 적정 거리 유지 시에만 입력 허용 (물리적 기상 유도).

최종 종료: 소음 센서 기반의 알람 종료 및 환경 정보(날짜, 온습도) 브리핑.

2. 기술 스택
언어: Python

하드웨어: PYC Basic (Raspberry Pi 4), Soda OS

구성: OLED, PiezoBuzzer, PSD, Gesture 센서, Sound 센서, LED 모듈, 스위치 등

주요 라이브러리:

Threading: 알람음, 제스처 인식, 비밀번호 입력의 병렬 처리.

pop: 하드웨어 추상화 계층(HAL)을 통한 센서 및 액추에이터 제어.

Random/Time: 정답 생성 및 비밀번호 점멸 좌표 계산, 타이머 로직.

개발 환경: JupyterLab (실시간 센서 데이터 모니터링 및 디버깅)

3. 시스템 아키텍처
A. Setup Phase: 스위치를 활용한 시간 설정 (OLED 디스플레이 UI).

B. Wait Phase: 설정된 시간까지 운영 타이머 가동.

C. Mission Phase (Core): 알람음 연주와 함께 3가지 미션 수행.

비밀번호 입력: PSD 센서로 적정 거리를 유지하며 점멸하는 4자리 번호 입력.

스누즈(Snooze): 좌→우 제스처 인식 시 5분 연장 (최대 3회).

LED 가이드: 스레드를 이용한 시각적 사용자 유도.

D. Final Phase: 소음 센서를 통한 최종 종료 및 온습도 브리핑.

4. 핵심 기술 포인트
비동기 멀티태스킹 기반의 사용자 경험(UX) 설계

알람음, 입력 감시, 제스처 인식, LED 가이드를 4개의 독립적인 스레드로 병렬 실행하였습니다.

플래그(Flag) 기반의 생명주기 관리를 통해 스레드 간 충돌을 방지하고 메인 스레드와의 동기화를 최적화했습니다.

실시간 센서 디버깅 및 수치 최적화

JupyterLab의 대화형 환경을 통해 센서 원시 데이터를 모니터링하고 적절한 임계값을 도출했습니다.

평균값 필터링을 적용하여 일시적으로 튀는 데이터에 의한 오작동을 방지하는 안전장치를 마련했습니다.

5. 트러블 슈팅
문제 1: 여러 스레드 동시 실행 시 CPU 점유율 과부하 및 중복 실행

해결: 루프 조건을 alarm_on으로 통일하고, 루프 내 첫 줄에 조건문을 추가하여 불필요한 연산 시 즉시 continue 하도록 설계해 부하를 최소화했습니다.

문제 2: PSD 센서 안내와 비밀번호 입력 UI 렌더링 충돌

해결: 시각적 노이즈 해결을 위해 우선순위 기반의 화면 갱신 로직을 적용, 사용자에게 가장 필요한 정보(거리 안내 등)가 먼저 출력되도록 제어했습니다.

6. 프로젝트 회고
파이썬 멀티스레딩을 하드웨어에 접목하며 변수 제어와 동기화 조건 정립의 중요성을 확인했습니다. 하드웨어 특성상 발생하는 병목 현상을 해결하기 위해 타이밍 최적화에 집중하였으며, 기획 단계의 체계적인 설계가 개발 비용(시행착오)을 줄이는 핵심임을 깊이 깨달았습니다.
