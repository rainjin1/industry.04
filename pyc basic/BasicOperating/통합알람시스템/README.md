🚨 미션 알람 시스템
1. 📌 프로젝트 개요
🎯 목표

단순히 소리만 울리는 알람이 아닌,
사용자가 실제로 잠에서 깰 수 있도록 행동을 요구하는 미션 기반 하드웨어 알람 시스템 구현

⚙️ 주요 기능
OLED + 가변저항 + 스위치를 활용한 시간 설정 UI
delay 한계를 극복한 비동기 기반 알람 + 미션 동시 수행
제스처 센서를 활용한 스누즈(Snooze) 기능 (최대 3회)
PSD 적외선 거리 센서를 이용한 입력 가능 거리 제한
소음 측정을 통한 알람 종료 + 날짜 및 환경 정보 브리핑

3. 🏗️ 시스템 아키텍처
A. Setup Phase
스위치를 활용한 직관적인 시간 설정
OLED 디스플레이 UI 제공
B. Wait Phase
설정된 시간까지 대기
C. Mission Phase (Core)

알람 발생과 동시에 3가지 미션 수행

1) 🔐 비밀번호 입력
PSD 센서로 거리 유지
OLED에 표시되는 점멸 4자리 비밀번호 입력
2) ✋ 스누즈 (Snooze)
제스처 센서 (Left → Right)
5분 연장 (최대 3회)
3) 💡 LED 가이드
스레드를 활용한 시각적 사용자 유도
D. Final Phase
소리 감지 센서를 이용한
👉 “소리 질러 알람 끄기”
이후 날짜 및 환경 정보 브리핑 제공

5. 🐛 트러블 슈팅
❗ 문제 1: 높은 CPU 점유율 & 스레드 중복 실행

원인

여러 스레드가 동시에 실행되며 과부하 발생

해결

모든 스레드의 while 조건을 alarm_on으로 통일

내부에 개별 조건 추가:

if not condition:
    continue
불필요한 연산 최소화
스레드 실행 조건 정교화로 중복 실행 제거
❗ 문제 2: OLED UI 충돌 (거리 안내 vs 비밀번호 입력)

원인

여러 센서 출력이 동시에 디스플레이에 반영되며 충돌 발생

해결

우선순위 기반 화면 갱신 로직 적용
시각적 노이즈 제거 및 UI 안정성 확보
6. 📖 프로젝트 회고

파이썬의 멀티스레딩을 실제 하드웨어에 적용하면서
스레드 제어와 변수 관리의 중요성을 깊이 이해하게 되었습니다.

특히:

소프트웨어와 달리 하드웨어는 병렬 처리 시 병목 현상이 빈번
각 센서와 출력 장치의 타이밍 제어가 핵심
변수 상태 관리 및 ON/OFF 타이밍을 더욱 정교하게 설계해야 함

또한:

JupyterLab을 활용하여 디버깅 시간을 크게 단축
실험 기반 개발을 통해 많은 시행착오 경험

👉 중요한 인사이트:

실제 환경에서는 시행착오 = 비용
따라서 기획 단계에서 더 정교한 설계가 필요함을 체감
