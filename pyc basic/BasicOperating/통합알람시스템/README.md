장난스러운 말투를 거두고, 제공해주신 원본의 전문적인 톤을 그대로 유지하면서 가독성 있게 마크다운(Markdown) 형식으로 정리해 드립니다.

각 항목을 명확하게 구분하고 기호와 들여쓰기를 활용하여 스캔하기 좋게 구성했습니다.

미션 알람 시스템 (Smart Mission Alarm)
1. 프로젝트 개요
목표: 소리만 울리는 단순한 알람이 아닌, 사용자가 완전히 잠에서 깰 수 있도록 행동을 요구하는 미션을 결합한 하드웨어 구현.

주요 기능:

OLED와 가변저항, 스위치를 사용한 시간 설정.

delay의 한계를 벗어나기 위해 비동기 함수를 이용한 알람음 연주 및 비밀번호 입력 미션 병렬 처리.

제스처 센서를 사용하여 최대 3회의 스누즈(Snooze) 기능 제공.

PSD 적외선 거리 센서를 이용한 입력 가능 범위 제한 (너무 가깝거나 멀 경우 입력 불가).

최종 소음 측정을 통한 알람 종료 및 날짜, 환경 정보 브리핑.

2. 기술 스택
프로그램 언어: Python

하드웨어: PYC Basic (라즈베리파이4), Soda OS 기반

구성: OLED, PiezoBuzzer, PSD, Gesture 센서, Sound 센서, LED 모듈, 스위치 등

Libraries:

Threading: 알람음, 제스처 인식, 비밀번호 입력을 병렬 처리하기 위함.

pop: 하드웨어 추상화 계층(HAL) 라이브러리를 통한 센서 및 액추에이터 제어.

Random, Time: 정답 생성, 비밀번호 점멸을 위한 랜덤 좌표 및 타이머 로직 구현.

개발환경: JupyterLab (인터랙티브 코드 테스트 및 실시간 센서 디버깅)

3. 시스템 아키텍처
A. Setup Phase: 스위치를 활용한 직관적인 시간 설정 (OLED display UI)

B. Wait Phase: 운영 타이머 가동

C. Mission Phase (Core): 알람음 연주와 동시에 3가지 미션 수행

비밀번호 입력: PSD 센서로 적정 거리를 유지하며 디스플레이에 점멸하는 4자리 비밀번호를 입력

스누즈(Snooze): 제스처 센서(Left → Right) 인식 시 타이머 5분 연장 (최대 3회)

LED 가이드: 스레드를 이용한 시각적 사용자 유도

D. Final Phase: 소음 센서를 통한 “소리 질러 알람 끄기” 및 날짜, 온습도 브리핑

4. 핵심 기술 포인트
비동기 멀티태스킹 기반의 사용자 경험(UX) 설계

알람음, 비밀번호 입력 감시, 제스처 인식, 시각적 스누즈 가이드를 4개의 독립적인 스레드로 병렬 실행하였습니다.

스레드 간 충돌 방지를 위해 플래그 기반으로 생명주기를 관리하였으며, 각 워커 스레드 내 조건부 로직을 설계하여 자원 낭비를 방지하고 메인 스레드와의 동기화를 최적화했습니다.

주피터랩을 활용한 실시간 센서 디버깅 및 수치 최적화

대화형 환경을 통해 센서의 원시 데이터를 실시간 모니터링하고, 축적된 데이터를 바탕으로 적절한 임계값을 설정했습니다.

평균값을 측정하여 일시적으로 튀는 값이 오작동을 일으키지 않도록 안전장치를 마련했습니다.

5. 트러블 슈팅
문제 1: 여러 스레드 동시 실행 시 CPU 점유율 과부하 및 중복 실행 발생

해결: 모든 스레드의 루프 조건을 alarm_on으로 통일하되, 루프 내 첫 줄에 개별 조건문을 추가하여 필요 없는 구간은 즉시 continue 되도록 설계하여 부하를 최소화했습니다. 중복 실행 문제는 정교한 조건 설정을 통해 디버깅했습니다.

문제 2: PSD 센서 안내와 비밀번호 입력 UI 렌더링 충돌

해결: 다양한 센서 피드백이 OLED에 동시 출력될 때 발생하는 시각적 노이즈를 해결하기 위해, 우선순위 기반의 화면 갱신 로직을 적용했습니다.

6. 프로젝트 회고
파이썬의 멀티스레딩 개념을 실제 하드웨어에 접목해보며 변수 제어와 스레딩 조건 정립의 중요성을 확인했습니다. 하드웨어 특성상 병렬 동작 시 발생하는 병목 현상을 해결하기 위해 변수 조절과 타이밍 최적화에 집중했습니다. 주피터랩을 통해 디버깅 시간을 단축할 수 있었으며, 이번 경험을 통해 실제 현장에서는 시행착오가 비용으로 직결되는 만큼 기획 단계부터 체계적인 설계가 선행되어야 함을 깊이 느꼈습니다.
